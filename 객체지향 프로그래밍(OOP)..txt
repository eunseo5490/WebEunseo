객체지향 프로그래밍(OOP).
객체지향이 아닐 때의 문제점
- 데이터와 기능이 따로 논다.
- 코드가 많아지게 되며 관리가 어렵다
- 재사용이 힘들다

객체지향의 핵심 개념
1. 클래스(class)
- 설계도
ex) 이 프로그램에서 사람을 어떻게 정의할 건지, 제품을 어떻게 정의할건지
클래스 안에 정의할 수 있는 내용
- 필드(field) : 각 개체가 가지는 정보(상태, 속성)
ㄴ 이름, 나이, 키, 몸무게, 혈액형, 성별, 주소, 학번 ...
- 메서트(method) : 각 객체가 할 수 있는 동작(기능)
ㄴ 공부한다, 먹는다, 운동한다, 잔다 ...

2. 객체(object)
클래스명 변수명 = new 클래스명();
- 클래스를 통해서 만들어진 것
- 필드에 객체만의 정보를 넣거나 필드의 내용을 사용할 수 있다.
객체변수명.필드명
- 클래스에 정의된 기능을 호출하여 사용할 수 있다.
객체변수명.메서드명();

3. 객체지향의 4대 특징
- 캡슐화
데이터를 보호하는 개념
필드 직접 접근하지 못하게 한다.
제공된 메서드를 통해서만 접근할 수 있게 한다.

- 상속
기존 클래스의 내용을 물려받는 기능(필드, 메서드)
코드를 재사용할 때 좋음

- 다형성
하나의 타입으로 여러 객체를 다룰 수 있는 성질
타이어마다 성능이 다르다.

- 추상화
클래스를 단순화 시키는 것

메서드 내부에서 메서드를 정의할 수 없다.
메서드 외부에서 메서드를 호출하는 것은 불가능하다.
