선언형 (결과중심)
- 메서드를 호출해서 결과만 확인하는 방식

왜 Stream API를 사용하면 좋은가?
- 코드가 짧아진다.
- 의도가 분명해진다.
- 중간 처리 과정이 체계적으로 정리된다.

핵심 특징
1. 원본 데이터를 직접 바꾸지 않는 경우가 많다.
2. 데이터 흐름을 중심으로 작성한다.
3. Stream 객체는 일회용이다.
- 한 번 사용한 객체는 다시 사용할 수 없다.
4. 지연 연산이 가능하다
- 결과를 바로 주는게 아니라 나중에 돌려주는 작업

Stream API 사용방법
1. 스트림 객체를 생성한다.
- 컬렉션이나 배열을 통해 스트림을 생성
2. 스트림의 데이터를 가공한다.
- 개수를 세거나, 더하거나, 필터링하거나, 변환하거나, 중복을 제거하는 등등
- 가공은 여러 개를 이어서 사용할 수 있다.

가공된 결과를 돌려준다.

컬렉션.stream()
  .중간연산()
  .중간연산()
  .중간연산()
  .최종연산();
