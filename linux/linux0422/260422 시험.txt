# 이론

1. 운영체제(OS)의 구조와 역할
운영체제는 하드웨어를 효율적으로 관리하고 사용자에게 편리한 환경을 제공하는 중재자입니다.

운영체제의 5대 핵심 관리 기능
	1. CPU (프로세스 관리) : 여러 프로그램이 동시에 실행되도록 조정한다.
	2. RAM (메모리 관리) : 각 프로그램에 필요한 공간을 할당하고 보호한다.
	3. 파일 시스템 관리 : 저장장치의 데이터를 생성, 삭제, 수정합니다.
	4. 입출력(I/O) 관리 : 키보드, 모니터, 네트워크 등 장치를 제어합니다.
	5. 네트워크 관리 : 다른 시스템과 데이터를 주고받을 수 있게 합니다.

운영체제의 종류
	데스크탑
	- Windows

쉘(Shell)과 터미널(Terminal)
	- 쉘 : 명령어를 해석하고 실행해주는 프로그램
	- 터미널 : 사용자가 글자를 입력할 수 있는 창(윈도우)


2. 윈도우와 리눅스의 주요 명령어 비교

패키지 관리자
리눅스는 텍스트 기반이라 웹에 직접 접근할 수 없기 때문에 패키지 관리자가 저장소에 접근하여 프로그램을 직접 다운받아준다.
리눅스에 프로그램을 다운받는 방법 : 
sudo apt install [프로그램명]

배포(WAS를 이용한 클라우드 배포) 사용 이유
- 내 컴퓨터에 서비스를 만든다. (로컬 환경)
- 다른 사람들은 내 IP주소를 모른다. (포드 설정)
- 외부로 노출이 되는거다보니 보안과 네트워크에 대한 위험성이 존재한다.
컴퓨터를 빌려서 내 서비스를 집어넣고 사람들이 접속하도록 하는 것
원격 컴퓨터의 IP주소를 알려줘서 다른 사람들이 접속하게 하면 된다.

웹서버(WAS) : 톰캣, Nginx
Nginx : 사용자의 요청을 받아서 서버로 전달해주는 중간 관리자
설치 명령어 : sudo apt install nginx


리눅스에 설치된 프로그램을 실행하는 방법
어떤 종류의 프로그램인지에 따라 다르다.

1. 실행ㅎ파일을 직접 실행
명령어 : ./[실행파일]

※ 리눅스는 실행 권한이 있어야 실행할 수 있다.
chmod +x [파일명]

2. 명령어로 실행(PATH 등록된 경우)
node
java
python3
/usr/bin같은 경로에 등록되어 있기 때문

3. 스크립트 실행(.sh 파일)
명령어 : bash [스크릷트파일].sh
- 스크립트는 리눅스에서 자주 쓰는 명령어들이 있을 때 하나로 묶는 것

얘네를 하나로 묶어야 한다.
cd project
npm install
npm run dev

아래처럼 묶는다.
start.sh


리눅스에서 프로그램에 우선운위 설정하기
-20 ~ 19 까지 존재하고 숫자가 낮을수록 우선순위가 높다.
일반 사용자는 낮추는 것만 가능하고 순위를 높이려면 관리자 권한이 필요하다.
nice : 프로그램을 실행하면서 우선순위를 결정
ㄴ nice -n 우선순위 명령
renice : 이미 실행중인 프로세스의 우선순위를 결정
