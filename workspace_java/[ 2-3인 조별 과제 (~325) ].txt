[ 1 변수의 종류 ]
참고자료(URL) : https://j-growthdiary.tistory.com/entry/%EB%B3%80%EC%88%98%EC%9D%98-%EC%A2%85%EB%A5%98%ED%81%B4%EB%9E%98%EC%8A%A4-%EB%B3%80%EC%88%98Static-%EB%B3%80%EC%88%98%EC%9D%B8%EC%8A%A4%ED%84%B4%EC%8A%A4-%EB%B3%80%EC%88%98%EC%A7%80%EC%97%AD-%EB%B3%80%EC%88%98

1) 클래스 변수 class variable (= static 변수)
- 정의 : 클래스 영역에서 타입 앞에 static이 붙는 변수
- 설명 : 객체를 공유하는 변수로 여러 객체에서 공통으로 사용하고 싶을 때 정의
- 선언위치 : 클래스 내부 (접근 제어자에 따라 바뀜)
	※ 접근 제어자 Access Modifier : 데이터를 보호하기 위해 접근 범위를 제한하는 것 (public, protected, default, private)
- 접근방법 : 클래스명.클래스변수명
	※ 클래스 변수는 객체를 생성하지 않아도 클래스명으로 바로 접근이 가능함

2) 인스턴스 변수 instance variable
- 정의 : 클래스 영역에서 static이 아닌 변수
- 설명 : 개별적인 저장 공간으로 객체/인스턴스마다 다른 값 저장 가능, 객체(=인스턴스)를 여러 개 생성했다면 각각의 참조 변수명으로 접근/값 저장 가능
- 선언위치 : 클래스 내부 (접근 제어자에 따라 바뀜)
- 접근방법 : 객체 생성 후 참조 변수명으로 접근

3) 지역 변수 local variable
- 정의 : 메서드 내에서 선언되고 메서드 수행이 끝나면 소멸되는 변수
- 설명 : 초기값을 지정한 후 사용할 수 있음 ex) int i = 0;
	※ 조건문, 반복문 내에 선언한 지역변수는 수행이 끝나면 소멸, 따라서 밖에서 선언해야 함
- 선언위치 : 메서드 영역 (접근 제어자 사용 X)

4) 매개 변수 (=parameter)
- 정의 : 메서드 호출 시 '전달하는 값'을 가지고 있는 변수
	※ 지역 변수처럼 선언된 곳부터 수행이 끝날 때까지 유효함
- 선언위치 : 메서드 영역 (접근 제어자 사용 X)

5) 자료형에 따라 기본자료형과 참조자료형으로 구분

[ 2 자바의 메모리 구조 ]
참고자료(URL) : https://lucas-owner.tistory.com/38

1) JVM이란?
- Java Virtual Machine(자바 가상 머신)의 약자로, 자바 프로그램이 실행될 수 있는 가상의 컴퓨터(가상 머신)
- .java → 컴파일러 → .class(=바이트코드)로 변환 → 클래스로딩(JVM ClassLoader) → 바이트코드 검증 → 실행 준비(Linking) → 실행(Execution)

2) 자바의 메모리 영역
- JVM은 OS로부터 메모리를 할당 받은 후 자바 프로그램에 맞게 여러 개의 영역으로 나누어 사용
- 각 영역은 어플리케이션의 성능에 영향을 준다. 같은 기능을 하는 어플리케이션일지라도, 메모리 관리에 따라 성능에 차이가 생긴다. 메모리를 관리하지 않고 구성하게 된다면 StackOverFlow가 발생하여 어플리케이션이 종료되거나, 혹은 어플리케이션의 속도가 크게 저하 될 수 있다.
- JVM의 메모리 영역은 크게 3가지로 이루어져 있다.
	1. Static 영역에는 클래스 정보와 static 변수가 저장되며 프로그램 종료 시까지 유지된다.
	2. Heap 영역에는 객체가 저장되고, Stack 영역에는 지역변수와 참조값이 저장되며 메서드 종료 시 제거된다.

!) JVM의 RunTime Data Area(런타임 데이터 영역)의 종류
1. 메서드 영역 Static Area(Method Area)
2. 힙 영역 Heap Area
3. 스택 영역 Stack Area
4. PC 레지스터 PC Register
5. 네이티브 메서드 스택 Native Method Stack
