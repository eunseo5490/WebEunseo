데이터 모델링
- 현실 세계의 업무 내용을 데이터베이스에 저장할 수 있도록 데이터 구조로 설계하는 과정

모델링이 필요한 이유
이상현상의 발생을 막아야 한다.
1. 데이터 중복이 발생할 수 있다.
2. 갱신 이상 -> 같은 정보가 여러 개 저장되어 있을 때 전부 수정하지 않아서 데이터의 불일치가 일어나는 현상
3. 삭제 이상 -> 필요 없는 데이터를 지우려다 필요한 데이터까지 삭제하는 경우
4. 삽입 이상 -> 필요한 데이터를 추가하려다 필요없는 데이터까지 추가하는 경우


데이터 모델링 과정

1. 요구사항 분석
- 업무에서 어떤 데이터를 관리해야 하는지 파악하는 단계
- 주요 명사를 찾아서 -> 엔티티로 만든다.


2. 개념적 모델링
- 업무 내용을 큰 그림으로 표현하는 단계이다.
- 주로 ERD를 사용한다.

주요 구성 요소
엔티티(Entity) : 관리해야 할 대상
속성(Attribute) : 엔티티가 가진 정보
관계(Relationship) : 엔티티 간 연결
식별자(Identifier) : 데이터를 구분하는 값

엔티티 간 관계 설정
관계를 문장으로 처리
- 학생은 수강신천을 한다.
- 수업은 수강신청을 받는다.
- 강사는 수업을 담당한다.

관계 차수 정하기
1ㄷ1, 1ㄷN, MㄷN 중 어떤 관계인지 정한다.
- 학생 1명은 여러 수강신청을 할 수 있다. -> 학생(1):수강신청(N)
- 수업 1명은 여러 수강신청을 받을 수 있다. -> 수업(1):수강신청(N)
- 강사 1명은 여러 수업을 담당할 수 있다. -> 강사(1):수업(N)
학생(N):수업(M)이 된다.
실제 설게에서는 중간 엔티티인 수강신청으로 관계를 풀어줘야 함.
학생(1) : N [수강신청] N : 수업(1)

논리 모델링
- 개념적 모델을 실제 테이블 구조에 가깝게 바꾸는 단계
- ERD를 가지고 테이블을 설계
- 테이블의 이름이나 속성의 이름을 정하는 단계
- 제약조건(PK,FK 등), 정규화


정규화란
실무에서는 보통 3차까지 다룸 -> 더 하면 오히려 조회속도가 느려짐

1NF(1차) -> 속성 하나에는 하나의 값만 들어가야 한다.
```
학생번호		이름	휴대폰 번호
1		김민수	010-1111, 010-2222
2		이영희	010-3333
```

제2정규형(2NF) -> 복합키에서 일부 컬럼에만 의존하는 컬럼을 제거(부분 함수 종속)
```
학생번호	과목번호	이름	과목명	성적
1	101	김민수	JAVA	A
1	102	김민수	SQL	B
```
학생번호 + 과목번호 -> 성적
학생번호 -> 이름
과목번호 -> 과목명

관련 있는 것들끼리 쪼갠다.
학생
학생번호		이름
1		김민수

과목
과목번호		과목명
101		JAVA
102		SQL

수강내역
학생번호		과목번호		성적
1		101		A
1		102		B
```

제3정규형(3NF) -> 일반 컬럼이 다른 일반 컬럼을 결정하면 안된다. (이행 함수 종속)
```
학생번호	이름	정공번호	전공이름
1	김민수	10	컴공
2	이영희	20	디자인

[ 중간에 다른 컬럼을 거쳐서 결정되면 분리 ]

학생 테이블
학생번호	이름	전공번호(PK)
1	김민수	10

전공 테이블
전공번호(FK)	전공이름
10		컴공
```

BCNF(Boyce-Codd Normal Form)
- 제 3규형을 좀 더 엄격하게 적용한 정규형
- 모든 함수적 종속에서 결정자는 반드시 후보키여야 한다.
※ 결정자 : 관계형 데이터베이스에서 다른 속성의 값을 유일하게 결정짓는 속성들의 집합을 의미한다.(A->B)
※ 후보키 : PK가 될 수 있는 속성

제3정규형을 지켜도 이상 현상이 남아있을 수 있다.
```
학생명	과목	강사
김민수	DB	이강사
김민수	OS	박강사
이영희	DB	이강사
```
subject가 결정자인데 후보키가 아님

과목```
과목명(PK)	강사
DB		이강사
OS		박강사
```
학생```		
학생명	과목
김민수	DB
김민수	OS
이영희	DB
```

제4정규형(4NF) -> 다중값 종속 제거
한 엔티티가 서로 독립적인 여러 값 집합을 가지면 분리
```
학생명		기술		취미
김민수		JAVA		축구
김민수		JAVA		게임
김민수		PYTHON		축구
김민수		PYTHON		게임

학생 기술
학생명		스킬
김민수		JAVA
김민수		PYTHON

학생 취미
학생명		취미
김민수		축구
김민수		자바
```

제5정규형(5NF) : 조인으롤 복원 가능한 경우 분리
- 어떤 테이블이 여러 개의 테이블로 분해되었을 때 JOIN을 하면 항상 원래 테이블이 나올 수 있는지
```
학생명	과목	강사
김민수	DB	이강사
김민수	OS	박강사
```
학생```
이름	과목
김민수	DB
```
강사```
과목	강사
DB	이강사
```
담당선생님```
학생명	강사
김민수	이강사
```


물리모델링
- 논리적 모델을 실제 DBMS에서 실행 가능한 SPL로 바꾸는 것
1. 컬럼 타입 구체적으로 정하기
2. NULL 허용 여부 저하기
3. 중복 불가 조건 정하기
4. 긱본값 정하기(DEFAULT)
5. 외래키 제약조건 정하기


DML(Data Manipulation Language) : 데이터 조작어
- 데이터에 대해서 조회, 수정, 추가, 삭제하는 명령어

SELECT [컬럼명1], [컬럼명2], ... FROM [테이블명];

모든 내용을 조회하고 싶다.
SELECT * FROM [테이블명]

특정 조건에 맞는 데이터 조회
SELECT [컬럼명] FROM [테이블명] WHERE [조건];

DB에서
= : 같다
<>, != : 같지않다

조건 여러 개
AND : 그리고
OR : 또는

NULL값이 들어있는 데이터를 조회
IS NULL
NULL값이 없는 데이터를 조회
IS NOT NULL

날짜 비교
here_date >= '1990-01-01'

BETWEEN A AND B : A와 B의 사이

LIKE
- 문자열 안에서 특정 글자나 패턴을 찾을 때 사용하는 조건문
SELECT 컬럼명
FROM 테이블명
WHERE 컬럼명 LIKE 패턴

SELECT * FROM FILM WHERE TITLE = 'LOVE'
- 이 SQL은 제목이 정확히 LOVE인 영화만 찾는다.

LIKE는 포함 여부, 시작 문자, 끝 문자, 특정 글자 패턴을 찾을 수 있다.

% : 글자 수와 상관 없이 아무 문자나 여러 개 올 수 있다는 뜻
'A%' : A로 시작하는 데이터
'%A' : A로 끝나는 데이터
'%ㅁ%' : A를 포함하고 있는 데이터
'A%B' : A로 시작하고 B로 끝나는 값

_ : 문자 1개를 의미한다.
'A_' : A로 시작하고 전체 길이가 2인 값

DCL(Data Control Language) : 데이터 제어어
GRANT : 권한을 부여
REVOKE : 권한을 회수

DCL이 필요한 이유
- 모든 사용자에게 모든 권한을 주면 위험해진다.

MySQL에서는 사용자를 다음과 같이 구분한다.
'사용자명'@'접속자위치'
'teacher'@'localhost'

사용자 생성하는 법
create user

TCL(Transaction Control Language)
트랜잭션 : 하나의 작업 묶음

update account set balance = nalance - 1000 where id =1;
update account set balance = nalance + 1000 where id =2;
